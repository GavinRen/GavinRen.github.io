GPU和CPU的对比

Posted on 2017-01-06 | In 深度学习 | Visitors

前有AlphaGo击败前世界冠军李世石,后有神秘棋手‘Master’在网上击败众多中日韩围棋顶尖高手。人工智能变得越来越火，在人工智能领域，深度学习目前最为流行。一提到深度学习，使用传统的CPU服务器训练模型，少则几周多则几个月。而使用GPU训练模型则时间缩短好多。GPU和CPU有什么区别和联系呢。为什么深度学习更适合使用GPU呢？最近在实验室调研GPU和CPU区别联系，下面介绍下自己理解的GPU 和CPU的区别

CPU 和GPU的联系

CPU是中央处理器（Central Processing Unit）的缩写，是一台计算机的运算核心和控制核心。GPU是图形处理器（Graphics Processing Unit）缩写。GPU也是显卡的处理器。

CPU和GPU的区别

逻辑结构

CPU和GPU的逻辑结构如下图所示，Control 是控制器，由程序计数器、指令寄存器、指令译码器、时序产生器和操作控制器组成，指挥计算机的各个组件按照指令的功能要求协调工作的部件。ALU是算术逻辑单元，是专门执行算术和逻辑运算的数字电路。Cache是CPU内部缓存,Cache是位于CPU与内存之间的临时存储器，它的容量比内存小的多但是交换速度却比内存要快得多，主要是为了解决CPU运算速度与内存读写速度不匹配的矛盾。DRAM是内存。GPU 将大部分晶体管用作执行单元，而不像CPU 那样用作复杂的控制单元和缓存。

优点不同

有上图的CPU和GPU逻辑图不同可以看出，GPU中的晶体管大部分用来做控制器和Cache,ALU 算术逻辑单元较少，所以CPU擅长处理具有复杂计算逻辑和复杂数据依赖的计算任务。GPU的核数远超CPU，并且每个核中的ALU 较多，所以GPU适合进行计算量大高度并行化的计算任务，但是由于控制器较少，GPU不太擅长处理计算逻辑复杂的任务，同时Cache 较少，导致GPU对含有大量数据依赖的任务不太擅长，例如含有向无环图DAG Spark分布式计算框架。

深度学习

使用CPU训练深度学习模型速度比使用GPU慢太多，所以在训练深度学习模型时，大多采用GPU进行训练。这是因为，例如深度学习中卷积神经网络，是许多卷积运算和矩阵运算的组合。并且数据之间没有相互独立，没有依赖，计算逻辑简单，但是计算量大可以高度并行化，所以使用GPU进行训练速度较快。
