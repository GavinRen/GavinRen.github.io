GPU安装相关问题

发表于 2017-03-01 | 分类于 深度学习 | 阅读次数

最近实验室购买一块M40GPU,准备安装GPU版的TensorFlow ，查了查发现TensorFlow GPU版的安装比CPU版的复杂很多，就在想为什么要安装这些东西，这些东西有什么作用，就查了查相关知识。

首先要安装GPU版的TensorFlow 就需要先安装GPU驱动，CUDA和cuDNN。下面分开介绍

GPU驱动

驱动程序是硬件厂商根据操作系统编写的配置文件，是添加到操作系统中的一小块代码，其中包含有关硬件设备的信息。所以下载GPU驱动的时候会分系统下载。这种程序创建了一个硬件与硬件，或硬件与软件沟通的接口，经由主板上的总线或其它沟通子系统与硬件形成连接的机制，这样的机制使得硬件设备上的数据交换成为可能。

当服务器上安装GPU 时，驱动程序是一项不可或缺的元件，可以说没有驱动程序，GPU就无法工作。所以第一步要先安装GPU 驱动

GPU 监控

GPU 监控有许多开源工具，我们采用GPU 本身的nvidia-smi命令，这个只能执行这个指令才能显示，当然这是一种缺陷，实际用的时候想监控不可能打入这个指令，这个时候可以采用watch命令以周期性的方式执行给定的指令。-n参数表示间隔几秒

watch -n 1 nvidia-smi命令每隔一秒显示结果

CUDA

随着显卡的发展，GPU越来越强大，而且GPU为显示图像做了优化。在计算上已经超越了通用的CPU。如此强大的芯片如果只是作为显卡就太浪费了，因此NVidia推出CUDA，让显卡可以用于图像渲染和计算以外的目的（例如这里提到的通用并行计算。

CUDA是一种通用并行计算框架，该框架使GPU能够解决复杂计算问题，最适合CUDA的也是可以高度并行化设计的。

从它的作用来看是为了让GPU能够处理复杂的通用并行计算，也就是说深度学习框架TensoFlow是无法直接运行在GPU上，而是运行在CUDA之上

cuDDN

cuDNN 是 NVIDIA 公司专门为深度学习领域开发的 GPU 加速库,cuDNN是NVIDIA推出的深度学习中CNN和RNN的高度优化的实现。因为底层使用了很多先进的技术和接口（没有对外开源），因此比其他GPU上的神经网络库性能要高不少，目前绝大多数的深度学习框架都使用cuDNN来驱动GPU计算。

关系

TensorFlow是最上层的深度学习框架，cuDNN是用来进行GPU加速的CUDA包，最后会通过CUDA在GPU之上运行。
