MySQL之三大范式

发表于 2017-03-17 | 分类于 MySQL | 阅读次数

数据库的三大范式主要的是要求的是表的结构，并且三大范式是递进要求的

第一范式

表的列具有原子性，不可再分解，即列的信息，不能再分解。例如下表就不满足第一范式

第二范式

第二范式在第一范式的基础上建立的，数据库表中不存在非关键字对任意候选关键字段的部分函数依赖（部分函数依赖指的是存在组合关键字中的某些字段决定非关键字段的情况）

上图可以看出（学号，课名）是候选码，学号和课名是关键字，例如非关键字系名部分依赖于学号，这就是部分函数依赖，所以不符合第二范式，其实就是组合关键字中某一个关键字会影响其它字段，就不满足第二范式

第三范式

满足第三范式的前提是满足第二范式，不存在非关键字段对任一关键字段传递函数依赖

意思就是说，只能直接依赖，不能间接依赖，例如

学号 系别 系主任
110 通信 张三

这就不满足第三范式，因为系主任依赖于系别，系别依赖于学号，则系主任间接依赖于学号，则不满足第三范式
