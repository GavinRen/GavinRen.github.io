MySQL之事务处理

Veröffentlicht am 2017-03-17 | in MySQL | Visitors

事务处理用来维护数据库的完整性，它保证成批的MySql操作要么完全执行，要么完全不执行

并不是所有的存储引擎都支持明确的事务管理，例如MyISAM和InnoDB是两种最常使用的引擎。前者不支持，后者支持。所以mysql 的存储引擎一般使用InnoDB.

事务的四大特性

1. 原子性：一组事务，要么成功，要么撤回
2. 稳定性：有非法数据，事务撤回
3. 隔离性：事务独立运行。彼此透明
4. 持久性：确保提交的事务的结果或效果的系统出现故障的情况下仍然存在

事务管理的指令

事务：指一组SQL语句

回退：指撤销指定SQL语句的过程

提交：指将未存储的SQL语句结果写入数据库表

保留点：指事物处理中设置的临时占位符，可以对它发布回退

事务隔离级别

SQL标准定义了4种隔离级别，低级别的隔离级别一般支持更高的并发处理，并拥有更低的系统开销

第一级别（读取未提交内容）

  1. 所有事务都能看到其他未提交事务的执行结果
  2. 本隔离级别很少用于实际应用，
  3. 该级别容易引发脏读：读到了未提交的数据

第二级别（读取提交内容）

  1. 大多数默认隔离级别
  2. 一个事务职能看见已经提交事务所做的改变
  3. 这种格力级别带来的问题：不可重复读：在同一个事物中执行完全相同的select语句可能看到不一样的结果
  4. 导致不可重复读的原因：有一个交叉的事务有新的commit,导致了数据的改变；

第三级别（可重读）

  1. MySQL的默认事务隔离级别
  2. 它确保同一个事务的多个实例在并发读取数据时，会看到同样的数据行
  3. 带来的问题：幻读：当用户读取某一范围的数据行时，另一个事务又在该范围内插入了新行，当用户在读取该范围的数据行时，会发现有新的“幻影”行

第四级别（可串行化）

  1. 最高的隔离级别
  2. 通过强制事务排序，是指不可能相互冲突，从而解决幻读问题。简言之，它是在每个读的数据行上加上共享锁
  3. 在这个级别可能会导致大量的超时现象和锁竞争
