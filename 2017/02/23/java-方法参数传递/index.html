基本类型

java 有八种基本类型：char,boolean,byte,short,int,long,float,double。其他的基本上就是引用类型。

在定义基本类型变量时，值直接存在变量中，变量是存在局部变量表上，也就是常说的栈内存上。而定义引用类型时，变量中实际存的是对象实例在堆内存的地址。变量名存的也在栈内存中。

方法中的参数传递

方法中的传递可以分为两种类型的传递，一是基本类型的传递，二是引用类型的传递。不管哪一种传递，都可以看作是值传递，基本类型可以看作是把实参赋值给了方法中的形参，形参中存的也是值，只不过在方法中队形参的修改，不会影响之前的值，所以方法一不会改变num的值。引用类型可以看作是值传递，是因为引用类型就行传递的是对象的地址。方法二是在把a 赋值了一个新的对象，此时a存的地址已经发生变化，指向的是新的对象。但是这样操作不会影响原来的对象的值。第三种方法，a指向的还是原来的对象，并调用了对象自身的方法所以可以更改。内存中为：

总结

值传递其实不不会改变传进来的实参，而引用类型会让人有一种改了实参的错觉是因为：其实引用变量传进来的时候，传进来的是对象在堆内存的地址，这个不会发生改变，当传进来的赋值给方法中的局部变量时，局部变量指向同样的对象，调用对象本身的方法或者更改对象的成员变量时，会使对象发生改变。所以在调用原来实参的变量名时，指向的对象发生了改变，而实参中的地址没有发生变换。所以引用变量的传递也可以看做是值传递。
