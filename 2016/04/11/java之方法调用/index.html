java方法调用

当调用一个新方法时，主调例程的所有局部变量和当前位置需要有系统存储起来，否则被调用的新方法将会重写由主调例程的变量所使用的内存和新方法运行完之后不知道想哪里转移。这些变量会被存在指定的寄存器上。
当存在方法调用时，需要存储所有重要信息（寄存器上的值和返回地址），都要以抽象的方式存在一张“纸”上并被置于堆的顶部，然后控制转移到新方法。当方法返回时，它查看堆顶部的那张纸并复原所有的寄存器。然后进行返回转移。这些工作均由栈来完成。
栈帧适用于支持虚拟机进行方法调用和方法执行的数据结构。它是虚拟机运行时数据区的虚拟机栈的栈元素。栈帧存储了方法的局部变量表，操作数栈，动态链接和方法返回地址等信息。
当同时运行大量的方法，可能出现栈空间用尽的情况。发生这种情况通常是失控递归。
