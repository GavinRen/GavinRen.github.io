剑指offer之数据流中的中位数

发表于 2017-02-24 | 分类于 剑指offer | 阅读次数

问题

如何得到一个数据流中的中位数？如果从数据流中读出奇数个数值，那么中位数就是所有数值排序之后位于中间的数值。如果从数据流中读出偶数个数值，那么中位数就是所有数值排序之后中间两个数的平均值。

思路

求中位数，如果数据流中是奇数个，中位数则是排序之后位于中间的数字，如果是偶数个数值，则是排序之后中间两个数的平均值。所以思路很明显，需要对读进来的数据流进行排序。排序有很多种方法，像归并排序和快速排序的时间复杂度都很低。当然也有堆排序，这种方法比较复杂。但是再海量数据中比较实用

剑指offer之两个链表的第一个公共节点

发表于 2017-02-24 | 分类于 剑指offer | 阅读次数

问题

输入两个链表，找出它们的第一个公共结点。

思路

首先要明白如果两个链表具有公共结点，那从这个公共节点的所有的节点都是相同的，之前自己理解的是只是中间相同，后面可以不同。

根据这个特点可以采用堆从最后一个节点开始判断，这样的话时间复杂度为$o(n)$,但是空间复杂度较高。如果不采用堆栈的思路去做的话，可以先计算出每个链表的长度，让那个长度比较大的链表先走几步，然后再一起遍历，看节点是否相同，第一个相同的则是答案。

问题

思路

最简单的想法，则是每次遍历一个字符，然后判断字符串后面是否还有这个字符，如果没有的话则返回，时间复杂度为$o(n^2)$.第二种想法采用利用存储空间换时间，定义一个hashMap,扫描两次数组，第一次把字符出现的次数，存进hashmap中，二次扫描判断字符出现的次数。

剑指offer之丑数

发表于 2017-02-23 | 分类于 剑指offer | 阅读次数

问题

把只包含因子2、3和5的数称作丑数（Ugly Number）。例如6、8都是丑数，但14不是，因为它包含因子7。 习惯上我们把1当做是第一个丑数。求按从小到大的顺序的第N个丑数

思路

最简单最粗暴的就是遍历每个数字，判断其是否符合丑数的定义。时间复杂度较高，还要判断不是丑数的数字，判断条件所耗费的时间也久。第二种思虑就是可以丑数本身的规律，丑数等于前面的丑数乘以2，乘以3和乘以5得到的。关键丑数是如何排序的，假设前面已经有若干个已经排好序的丑数，然后根据之前的丑数乘以2，乘以3和乘以5，来判断最小的。
